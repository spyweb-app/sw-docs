## Why SpyWeb?

<CardGrid stagger>
	<Card title="Lua-Powered Hooks" icon="pencil">
		Customize every pipeline stage — from request headers to notification payloads — with simple Lua functions. Only define the hooks you need.
	</Card>
	<Card title="Browser Automation" icon="laptop">
		Built-in CDP support auto-detects Chrome, Edge, or Brave on your system. Scrape JavaScript-rendered pages with `cdp.launch()`.
	</Card>
	<Card title="Multi-Worker Concurrency" icon="rocket">
		Run multiple workers per job to process URL queues in parallel. Workers share state through a single Lua VM.
	</Card>
	<Card title="Programmable API Server" icon="setting">
		Define custom HTTP endpoints in `server/init.lua`. Build dashboards, webhooks, or proxy APIs with full access to your scraped data.
	</Card>
	<Card title="Built-in Testing" icon="approve-check-circle">
		Write `test_*` functions alongside your hooks. Each test runs in an isolated Lua VM with full access to production bindings.
	</Card>
	<Card title="Flexible Storage" icon="document">
		Choose between a zero-config key-value store (redb) or full SQLite with custom tables, JOINs, and external tool access.
	</Card>
</CardGrid>

## Quick Start

```bash
# Install
curl -fsSL https://spyweb.app/install.sh | bash

# Create a job
mkdir -p jobs/my-job
```

Create `jobs/my-job/config.toml`:

```toml
name = "Example Monitor"
url = "https://example.com"
selector = ".item"
fields = ["title:h2", "link:a@href"]
```

```bash
spyweb start
```
