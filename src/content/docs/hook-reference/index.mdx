Every scrape run flows through a series of **pipeline hooks** you can define in your Lua script. Each hook receives a `request`, `response`, or `items` table plus the shared `ctx`, and can modify data, skip stages, or inject side effects.

:::note
**All hooks are optional.** Omit any you don't need, SpyWeb runs fine with just a `config.toml`.
:::

## Key Rules

- Return the value (modified or not) to **continue** the pipeline
- Return `nil` or `false` to **drop/skip** (behavior varies per hook)
- Hook errors are **logged and skipped**, they never crash the job
- Only define the hooks you actually need. SpyWeb pre-detects which functions exist at startup and skips the processing logic entirely for any that are missing.

Every hook receives a per-cycle `ctx` table as its second argument. See [Context](/hook-reference/context) for the full reference.

## Pipeline Stages

The hooks execute in this exact order during each scrape cycle:

<div class="pipeline-container">
  <div class="pipeline-flow">
    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">before_fetch(request, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Modify URL, add headers, or return nil to skip this run entirely.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">override_fetch(request, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Custom fetching phase (bypasses built-in HTTP client if defined).</p>
    </div>

    <div class="pipeline-stage stage-background">
      <div class="stage-header">
        <span class="stage-name">[HTTP fetch]</span>
        <span class="stage-thread">background thread</span>
      </div>
      <p class="stage-desc">Automatic - uses request config from previous stage.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">after_fetch(fetch_result, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Inspect success or failure, mutate body on success, or return a synthetic response to recover from fetch errors.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">override_extract(response, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Custom extraction phase for JSON/XML (bypasses built-in CSS extraction if defined).</p>
    </div>

    <div class="pipeline-stage stage-background">
      <div class="stage-header">
        <span class="stage-name">[CSS extraction]</span>
        <span class="stage-thread">background thread</span>
      </div>
      <p class="stage-desc">Automatic - raw parser with DOM fallback.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">after_extract(items, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Batch filter/modify all items. nil or empty = no items.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">filter_item(item, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Per-item filter. <strong>Replaces</strong> built-in keyword filter if defined.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">before_store(items, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Last chance before DB insert. nil = skip store + notify.</p>
    </div>

    <div class="pipeline-stage stage-background">
      <div class="stage-header">
        <span class="stage-name">[dedup + insert]</span>
        <span class="stage-thread">background thread</span>
      </div>
      <p class="stage-desc">Automatic - atomic check-and-insert.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">before_notify(items, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Reshape or silence notifications. Items already stored.</p>
    </div>

    <div class="pipeline-stage stage-executor">
      <div class="stage-header">
        <span class="stage-name">before_webhook(payload, ctx)</span>
        <span class="stage-thread">executor thread</span>
      </div>
      <p class="stage-desc">Reshape or silence webhook POSTs. Full JSON payload.</p>
    </div>

    <div class="pipeline-stage stage-background">
      <div class="stage-header">
        <span class="stage-name">[notify + webhook]</span>
        <span class="stage-thread">background thread</span>
      </div>
      <p class="stage-desc">Automatic - desktop notification + webhook POST.</p>
    </div>
  </div>
</div>
